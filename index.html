Updating Build Numbers for iOS and Android Apps

On a project that I belong to, we needed a way to integrate our Mobile App development (using Xamarin) into our existing version numbering process - using Team City build agents running Windows Server and Powershell - to update the Info.plist file for the iOS app and the AndroidManifest.xml file for Android.

This willowtreeapps.com page has a terrific solution for updating build and version numbers within MsBuild. I decided not to use this approach as MsBuild's XmlPoke function seems to drop the !DOCTYPE node when updating the Info.plist file.

Updating the build version for an iOS app is trivial if you use the plist update command on Mac OSX, but a simple Powershell script could update the versions for both app platforms. If you're developing apps using Xamarin and want to integrate automatic build versioning into your build process, here's an example of a Powershell script to update an app's version and build number. Creating pages manually

If you prefer to not use the automatic generator, push a branch named gh-pages to your repository to create a page manually. In addition to supporting regular HTML content, GitHub Pages support Jekyll, a simple, blog aware static site generator. Jekyll makes it easy to create site-wide headers and footers without having to copy them across every page. It also offers intelligent blog support and other advanced templating features.
